# Mitochondrial Variant Classification Workflow

## Overview
This document describes the hierarchical process for classifying mitochondrial DNA variants detected in single-cell RNA sequencing data. The classification prioritizes reliability and biological significance through multiple filtering and categorization steps.

## Global Parameters

### Quality Control Thresholds
- **CUTOFF_MIN_READS**: 10 (minimum read depth required)
- **CUTOFF_MIN_CELLS**: 10 (minimum cells required per cell type)
- **CUTOFF_NOTRELIABLE**: 10 (minimum cells for reliable variant)
- **CUTOFF_HETEROPLASMIC**: 0.05 (minimum AF for heteroplasmic variant)
- **CUTOFF_HOMOPLASMIC**: 0.95 (minimum AF for homoplasmic classification)

### Somatic Variant Thresholds
- **CUTOFF_SOMATIC_IGNORE_N_CELLS**: 2 (minimum cells with variant to consider)
- **CUTOFF_SOMATIC_MIN_N_CELLS**: 10 (minimum cells with variant for reliable somatic call)

### Technical Exclusion Positions
- **POS_MISSALIGNMENT_ERROR**: Positions 66-71, 300-316, 513-525, 3106-3107, 12418-12425, 16182-16194
- **POS_RNA_EDITING**: Positions 585, 1610, 3238, 4271, 5520, 7526, 8303, 9999, 10413, 12146, 12274, 14734, 15896, 295, 2617, 13710

---

## Classification Pipeline

### Step 1: Filter Unreliable Variants

All variants are first screened to remove technically unreliable calls. A variant is marked as **Not Reliable** if it meets ANY of the following criteria:

#### 1.1 Mis-alignment Errors (`excluding_pos`)
- **Criteria**: Variant position is in `POS_MISSALIGNMENT_ERROR` regions
- **Reason**: These positions are prone to sequencing alignment artifacts
- **Positions**: 66-71, 300-316, 513-525, 3106-3107, 12418-12425, 16182-16194

#### 1.2 RNA Editing Sites (`editing`)
- **Criteria**: Variant position is in `POS_RNA_EDITING` sites
- **Reason**: RNA editing creates artificial variants that don't reflect DNA-level mutations
- **Positions**: 585, 1610, 3238, 4271, 5520, 7526, 8303, 9999, 10413, 12146, 12274, 14734, 15896, 295, 2617, 13710

#### 1.3 Low Cell Number (`n_cells`)
- **Criteria**: Variant detected in fewer than `CUTOFF_NOTRELIABLE` (10) cells
- **Additional filters applied**:
  - Cell-level AF ≥ `CUTOFF_HETEROPLASMIC` (0.05)
  - Cell-level depth ≥ `CUTOFF_MIN_READS` (10)
- **Reason**: Variants in too few cells are likely false positives or sequencing noise

**Output**: Variants passing all three filters proceed to biological classification

---

### Step 2: Remove Ethnicity-Related Variants (`haplo`)

After filtering unreliable variants, remove haplogroup-defining variants:

- **Criteria**: Variant has non-empty Haplogroup annotation
- **Reason**: These are germline polymorphisms defining mitochondrial lineages, not somatic mutations
- **Note**: This filter is applied independently and does not affect downstream homo/hete/somatic classification

---

### Step 3: Classify Homoplasmic vs Heteroplasmic Variants

Remaining variants are classified based on their allele frequency patterns across cell types.

#### 3.1 Calculate Cluster-Level Allele Frequencies
For each variant in each cell type (cluster):
- Aggregate all cell-level reads within the cell type
- Calculate cluster AF = (Alt reads) / (Total reads)
- Require cluster-level total reads ≥ `CUTOFF_MIN_READS` (10)

#### 3.2 Homoplasmic Classification (`homo`)
A variant is classified as **Homoplasmic** if it meets EITHER criterion:

**Criterion A**: High AF in all cell types with sufficient coverage
- Count cell types where cluster AF > `CUTOFF_HOMOPLASMIC` (0.95) → `n_celltypes_have_highaf`
- Count cell types with total reads ≥ `CUTOFF_MIN_READS` → `n_celltypes_have_reads`
- **Classify as Homoplasmic if**: `n_celltypes_have_highaf` ≥ `n_celltypes_have_reads`

**Criterion B**: High average AF across all cell types
- Calculate mean cluster AF across all cell types
- **Classify as Homoplasmic if**: mean cluster AF ≥ `CUTOFF_HOMOPLASMIC` (0.95)

**Interpretation**: Homoplasmic variants are present at near-fixation in all or most cells, indicating germline or early developmental mutations.

#### 3.3 Heteroplasmic Classification (`hete`)
A variant is classified as **Heteroplasmic** if:
- It passes the unreliable filter (Step 1)
- It is NOT classified as Homoplasmic (Step 3.2)

**Interpretation**: Heteroplasmic variants show variable AF across cells (0.05 < AF < 0.95), indicating potential somatic mutations or tissue-specific variants.

---

### Step 4: Identify Somatic Variants (Subset of Heteroplasmic)

Somatic variants are a **specialized subset of heteroplasmic variants** showing cell-type-specific patterns.

#### 4.1 Input
- Only heteroplasmic variants from Step 3.3
- Cell-level data with depth ≥ `CUTOFF_MIN_READS` (10)

#### 4.2 Cell Type Categorization
For each variant and cell type, categorize cells as:
- **"colorful"**: Cells carrying the variant (AF ≥ threshold)
- **"black"**: Cells with sufficient coverage but no variant (wild-type)

#### 4.3 Somatic Criteria
A heteroplasmic variant is classified as **Somatic** if ALL conditions are met:

1. **Presence in multiple cell types**:
   - At least one cell type has ≥ `CUTOFF_SOMATIC_IGNORE_N_CELLS` (2) cells with variant
   - At least one cell type has ≥ `CUTOFF_SOMATIC_MIN_N_CELLS` (10) cells with variant

2. **Absence in other cell types**:
   - At least one cell type has ≥ `CUTOFF_MIN_CELLS` (10) wild-type cells (black)

3. **Cell-type specificity**:
   - Number of cell types with wild-type cells > Number of cell types with variant cells + 1
   - Formula: `n_celltypes_have_reads - n_celltypes_have_variant > 1`

**Interpretation**: Somatic variants show clear cell-type-specific enrichment, suggesting they arose during development or tissue differentiation.

## Classification Hierarchy Summary

```
All Detected Variants
│
├─ Step 1: UNRELIABLE VARIANTS (Excluded)
│   ├─ Mis-alignment errors (excluding_pos)
│   ├─ RNA editing sites (editing)
│   └─ Low cell number (n_cells < 10)
│
└─ RELIABLE VARIANTS
    │
    ├─ Step 2: Ethnicity-related (haplo) [Tracked separately]
    │
    └─ Step 3: BIOLOGICAL CLASSIFICATION
        │
        ├─ HOMOPLASMIC (homo)
        │   • AF ≥ 0.95 in all cell types, OR
        │   • Mean AF ≥ 0.95 across cell types
        │
        └─ HETEROPLASMIC (hete)
            • 0.05 ≤ AF < 0.95
            │
            └─ Step 4: SOMATIC (subset of hete)
                • Cell-type specific pattern
                • Present in ≥10 cells in at least one cell type
                • Absent in ≥10 cells in at least one other cell type
                • Shows clear differential distribution
```

---

## Output Categories

The final classification produces these mutually exclusive (except somatic ⊂ hete) categories:

1. **excluding_pos**: Mis-alignment error positions
2. **editing**: RNA editing sites
3. **n_cells**: Low cell count variants
4. **haplo**: Ethnicity/haplogroup defining variants
5. **homo**: Homoplasmic variants (AF ≥ 0.95)
6. **hete**: Heteroplasmic variants (0.05 ≤ AF < 0.95)
7. **somatic**: Cell-type-specific heteroplasmic variants (subset of hete)

## Notes for Visualization

When generating flowcharts or diagrams:

1. Use decision diamonds for filtering steps (Steps 1, 2)
2. Use process boxes for calculation steps (Step 3.1)
3. Use conditional branches for classification logic (Steps 3.2, 3.3, 4.3)
4. Highlight that **somatic is a subset of heteroplasmic**, not a separate category
5. Show thresholds and parameters on decision branches
6. Use different colors for:
   - Red: Unreliable/Excluded variants
   - Yellow: Ethnicity-related variants
   - Green: Homoplasmic variants
   - Blue: Heteroplasmic variants
   - Purple: Somatic variants (specialized heteroplasmic)
