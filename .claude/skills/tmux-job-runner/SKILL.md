# Tmux Job Runner

Create shell scripts that launch long-running R/Python analysis jobs in detached tmux sessions with structured logging, status tracking, and metadata capture.

## Workflow

1. **Implement the analysis script** (R or Python) following project conventions
2. **Create the tmux launcher** shell script in the output directory
3. **Submit** by running the launcher script
4. **Monitor** progress via log files and status checks

## Step 1: Create tmux launcher script

Place the launcher at: `{OUTPUT_DIR}/run-{script-short-name}-tmux.sh`

### Single-job template

Use `references/single-job-tmux.sh` as the template. Key structure:

```bash
#!/usr/bin/env bash
set -euo pipefail

# --- Configurable variables (with defaults) ---
REPO_DIR="${REPO_DIR:-/home/liuc9/github/scMOCHA-data}"
SCRIPT_PATH="${SCRIPT_PATH:-${REPO_DIR}/high-res/NN-script-name.R}"
RSCRIPT="${RSCRIPT:-/scr1/users/liuc9/tools/miniforge3/envs/renv/bin/Rscript}"
CONDA_SH="${CONDA_SH:-/scr1/users/liuc9/tools/miniforge3/etc/profile.d/conda.sh}"
CONDA_ENV="${CONDA_ENV:-renv}"
SESSION_PREFIX="${SESSION_PREFIX:-jobprefix}"
# ... script-specific args ...

# --- Derived paths (do not change) ---
OUTPUT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
TIMESTAMP="$(date '+%Y%m%d_%H%M%S')"
SESSION_NAME="${SESSION_PREFIX}_${TIMESTAMP}"
LOG_DIR="${OUTPUT_DIR}/logs-launch_${TIMESTAMP}"
LOG_FILE="${LOG_DIR}/${SESSION_NAME}.log"
STATUS_FILE="${LOG_DIR}/${SESSION_NAME}.status"
META_FILE="${LOG_DIR}/${SESSION_NAME}.meta"
JOB_SCRIPT="${LOG_DIR}/${SESSION_NAME}.sh"
```

### Parallel-jobs template

Use `references/parallel-jobs-tmux.sh` when running across multiple dimensions (e.g., celltype levels L1/L2). Loops over a dimension array and creates one tmux session per item, each with its own log/status/meta/job files under a shared `LOG_DIR`.

```bash
LEVELS_LIST=("L1" "L2")  # or DATASETS, COMPARISONS, etc.
for LEVEL in "${LEVELS_LIST[@]}"; do
  SESSION_NAME="${SESSION_PREFIX}_${LEVEL}_${TIMESTAMP}"
  # ... create LOG_FILE, STATUS_FILE, META_FILE, JOB_SCRIPT per session ...
  tmux new-session -d -s "${SESSION_NAME}" "${JOB_SCRIPT}"
done
```

## Step 2: Job script internals (generated by launcher)

Every generated `JOB_SCRIPT` must include these sections in order:

```bash
#!/usr/bin/env bash
set -euo pipefail

# 1. Exit trap — writes exit code to .status file
on_exit() {
  local status=$?
  echo "[$(date '+%F %T')] session='SESSION_NAME' finish status=${status}"
  printf "%s\n" "${status}" > 'STATUS_FILE'
  exit ${status}
}
trap on_exit EXIT

# 2. Tee all output to log file
exec > >(tee -a 'LOG_FILE') 2>&1

# 3. Start banner
echo "[$(date '+%F %T')] session='SESSION_NAME' start"

# 4. Conda activation (with nounset guards)
set +u
source 'CONDA_SH'
conda activate 'CONDA_ENV'
set -u

# 5. cd to repo and run
cd 'REPO_DIR'
'RSCRIPT' 'SCRIPT_PATH' --arg1 val1 --arg2 val2
```

### Critical patterns

- **`set +u` / `set -u` around conda**: conda scripts reference unset variables — guard with `set +u` before `source` and `set -u` after `conda activate`
- **`escape_sq()` helper**: Use for all paths embedded in heredoc to handle single quotes safely:
  ```bash
  escape_sq() { printf "%s" "$1" | sed "s/'/'\\\\''/g"; }
  ```
- **Meta file**: Write key=value pairs capturing all launch parameters for reproducibility
- **Verbose flag**: Map `VERBOSE=true` to `--verbose` arg; empty string when false

### Preflight checks (single-job)

Include before launching tmux:
```bash
[[ ! -f "${SCRIPT_PATH}" ]] && echo "Missing R script" >&2 && exit 1
[[ ! -f "${CONDA_SH}" ]] && echo "Missing conda" >&2 && exit 1
command -v tmux >/dev/null 2>&1 || { echo "tmux not found" >&2; exit 1; }
tmux has-session -t "${SESSION_NAME}" 2>/dev/null && { echo "Session exists" >&2; exit 1; }
```

For parallel jobs, use `continue` instead of `exit 1` when a session already exists.

## Step 3: Submit

```bash
bash /path/to/run-script-name-tmux.sh
```

Or with overrides:
```bash
LEVELS="L1" NTHREAD=4 bash /path/to/run-script-name-tmux.sh
```

## Step 4: Monitor

### Check running sessions
```bash
tmux ls | grep SESSION_PREFIX
```

### Check completion status
```bash
cat /path/to/logs-launch_*/SESSION_NAME.status
# "0" = success, non-zero = error
```

### Check logs
```bash
# Last 20 lines
tail -20 /path/to/logs-launch_*/*.log

# Search for errors
grep -i "error\|fatal\|traceback" /path/to/logs-launch_*/*.log

# Count completed items (R logger pattern)
grep "INFO.*Completed" /path/to/logs-launch_*/*.log | wc -l
```

### Kill a running job
```bash
tmux kill-session -t SESSION_NAME
```

## Naming conventions

| Item              | Pattern                         | Example                              |
| ----------------- | ------------------------------- | ------------------------------------ |
| Launcher script   | `run-{short-name}-tmux.sh`      | `run-af-cutoff-comparison-tmux.sh`   |
| Parallel launcher | `run-parallel-tmux.sh`          | `run-parallel-tmux.sh`               |
| Session prefix    | `af{variant}_{scriptnum}`       | `af8362_1708`                        |
| Log directory     | `logs-launch_{YYYYMMDD_HHMMSS}` | `logs-launch_20260413_143022`        |
| Log file          | `{session_name}.log`            | `af8362_1708_20260413_143022.log`    |
| Status file       | `{session_name}.status`         | `af8362_1708_20260413_143022.status` |
| Meta file         | `{session_name}.meta`           | `af8362_1708_20260413_143022.meta`   |
| Job script        | `{session_name}.sh`             | `af8362_1708_20260413_143022.sh`     |

## Output directory convention

Tmux launchers live alongside their output:
```
high-res-MANUSCRIPTFIGURES-notuse/allvariants-prioritize/
  {script-name}/
    {variant}/
      run-{name}-tmux.sh          # launcher
      run-parallel-tmux.sh         # parallel launcher (if needed)
      logs-launch_{timestamp}/     # log directory per launch
        {session}.log
        {session}.status
        {session}.meta
        {session}.sh
      {output files from R script}
```
